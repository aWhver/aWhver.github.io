DOM:

1.子节点

元素.childNodes 元素节点是节点列表 有兼容性问题
需判断nodeType 节点类型
children 与childNodes 功能一样 无兼容性问题
元素.attributes 属性节点

2.父节点

parentNode 无兼容性问题

3.firstChild

标准下：文本节点 非标准（ie678）：只包含元素节点

4.firstElementChild

标准下：只包含元素节点 非标准：undefined（用来判断）

5.DOM元素的创建、添加、删除

createElement //创建一个节点 px:var oLi=document.createElement('li')
appendChild //在末尾追加一个节点 px：父级.appendChild（oLi）
insertBefore //在已有元素前面添加 px:父级.insertBefore（oLi,aLi[num]）（ie下报错）

兼容方法：

if（aLi.length>0）{
父级.insertBefore（oLi，aLi[num]）
}else
{
父级.appendChild（oLi）
}
removeChild //删除节点 父级.removeChild（删除谁）

6.offset

offsetTop：当前元素到定位父级的距离
如果没定位父级 1.标准下是到html的距离
2.非标准下到body的距离
offsetParent
没有定位父级 body
非标准下，自身有定位。没有定位父级 html
自身没有定位，没有定位父级 body
如果父节点有zoom，定位父级就指向有zoom的元素
clientWidth 元素内容的宽度 样式宽+padding
offsetWidth 样式宽+padding+border
scrollheight 滚动高度 toppadding+内容高度+margin

事件绑定：

ie678下 对象.attachEvent（事件名，事件函数）点击onclick此种绑定方式可以让同一个对象执行多种事情，类似函数封装
标准下 对象.addEventListener(不加on的事件名，事件函数，是否捕获)点击click 默认捕获true
冒泡事件 执行从下至上 捕获事件 执行从上至下

兼容方法：

function bind（obj,eName,fn）{
if (obj.addEventListener)//标准下
{
obj.addEventListener（eName,fn，false）
}else{//678
obj.attachEvent（”on”+eName，function（）{
       fn.call（obj）；
     }）
   }
}
