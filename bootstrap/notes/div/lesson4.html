< pre> </pre>

2创建文本移动标记-marquee

了解知识点 ：现在已经被w3c抛弃 代码多 没有语义
a1、默认向方向 direction="left" right右 down下 up上 来回走alernate
a2、文本背景颜色 bgcolor="red"
a3、scroll 单方向滚动 默认向左
a4、loop 循环次数 loop="自定义次数" 如果true 无限循环
a5、scrollAmount="速度值" 文本速度值
a6、鼠标移入onmouseover 停止 onmouseover=this(指的是marquee当前标记).stop() 停止
鼠标移入marquee让它自己停止动画
a7、鼠标移出onmouseout 在执行动画 动起来 onmouseout=this.start() 开始动

3文本域标记

< textarea> </textarea>
作用：1、文本输入 input type="text"(信息 密码账号 )也可以
2、存储数据
控制输入文本域大小： < textarea rows="400" cols="400"> 不是像素
rows 高度== 行合并
cols 宽度= 列合并
实际大小 字体大小字数*400
默认情况下
a1、resize 可以拖动 如需要禁止 在CSS样式里面添加 resize:none;
a2、默认可以输入内容 disabled="true" 禁止输入内容disabled="false"
a3、输入内容只读状态 readonly
div 也可以编辑状态 用到html5 contenteditable
输入状态有焦点 js里面的 focus 获取 change 修改文本内容 触发
按tab 输入域的遍历顺序 tabindex

5、 超链接标签

< a href="跳转页面的地址" target="跳转窗口"></a>
跳转窗口 默认 target="_self" 本页面窗口打开 target="_blank" 新窗口
a 锚点 第一创建锚点 name="锚点名字" id="名字"

6、内联框架

内联：例子 < div> < iframe src="加载内容的路径" frameborder="1"> </iframe> </div>
src 框架加载的内容路径 frameborder 框架的边框
外部链接 本地图片 网页都可以
优点; a 导入第三方内容 加载速度较快速
b 安全性强
c 加载脚本
缺点 a iframe会阻塞主页面的Onload事件
b. 即时内容为空，加载也需要时间
c. 没有语意
